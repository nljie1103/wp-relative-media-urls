# 九流媒体相对地址

一个面向 WordPress 反向代理、多入口域名、源站隐藏和缓存节点场景的轻量插件。

## 3.0.0 设计原则

- 默认不启用任何转换。
- 媒体库地址、文章内容地址、多域名站内链接分开控制。
- 历史文章必须扫描、预览、勾选后才会写入数据库。
- 媒体库输出转换默认是可逆的输出层转换，不移动文件、不删除文件、不修改 WordPress 核心。
- 多域名访问适配默认关闭，推荐使用白名单模式，避免任意 Host 参与生成站内链接。

## 功能模块

### 一、媒体库地址

- 转换已经上传媒体的输出地址。
- 从开启后，新上传媒体默认使用相对地址输出。
- 支持媒体库弹窗、编辑器、附件 URL、特色图、srcset 等输出层。

### 二、文章内容地址

- 保存文章/页面时自动转换内容中的媒体地址。
- 前台显示文章时临时转换旧内容。
- 扫描历史文章，预览并转换选中文章。

### 三、多域名访问适配

用于解决：通过香港反代域名进入首页后，点击文章/页面/菜单又跳回 WordPress 数据库中固定站点域名的问题。

- 支持站内链接跟随当前访问域名。
- 通过 `pre_option_home` / `pre_option_siteurl` 动态适配 WordPress 生成链接时使用的站点地址。
- 支持白名单域名模式。
- 支持 HTTPS / HTTP / 自动协议识别。
- 可选前台 HTML 兜底替换，把站内绝对链接改写到当前域名。
- 可排除后台、登录页、REST API、Feed、Sitemap、XML 等系统端点。

## 推荐使用方式

1. 安装并启用插件。
2. 进入“媒体相对地址”设置页。
3. 先填写“额外源站域名”和“多域名白名单”。
4. 先只开启“启用站内链接跟随当前访问域名”，测试首页点击文章是否还会跳回源站。
5. 确认无误后，再按需开启媒体库输出转换、前台临时转换或保存文章时转换。
6. 历史文章需要先扫描、预览，再手动勾选转换。

## 注意事项

- 插件不会删除图片，不会移动媒体库文件。
- 批量转换历史文章会写入 `post_content` / `post_excerpt`，执行前建议备份数据库。
- 多域名访问适配建议使用白名单模式。
- 如果启用了缓存插件、宝塔反代缓存或 Nginx proxy_cache，修改设置后请清理缓存再测试。
