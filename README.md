# 九流媒体相对地址

WordPress 反向代理与多域名链接助手。适用于：

- 美国源站运行 WordPress，香港/日本/新加坡 VPS 做反向代理缓存。
- WordPress 固定站点域名导致图片、文章链接、菜单链接跳回源站。
- 多入口域名共用一个 WordPress，需要前台链接跟随当前访问域名。
- 换域名、源站隐藏、HTTP/HTTPS 混合内容、旧媒体链接清理。

## 核心原则

- 默认不启用任何转换。
- 不移动媒体文件。
- 不修改 WordPress 核心。
- 不默认修改数据库。
- 所有永久修改必须手动扫描、预览、勾选、确认。
- postmeta 默认只读扫描，避免破坏序列化数据。

## 主要功能

### 媒体库地址

- 转换已上传媒体输出地址。
- 从开启后，新上传媒体默认相对地址输出。
- 支持附件 URL、媒体库弹窗、特色图、srcset。

### 文章内容地址

- 保存文章时自动转换媒体地址。
- 前台输出时临时转换旧文章内容。
- 历史文章扫描、Dry Run 预览、勾选转换。
- 支持恢复相对媒体 URL 为完整 URL。

### 多域名访问适配

- 动态适配 home/siteurl。
- 白名单域名模式。
- 文章、页面、分类、菜单等站内链接跟随当前访问域名。
- 前台 HTML 兜底替换。
- 可选根相对站内链接。
- 默认排除后台、登录、REST、Feed、Sitemap、XML。

### SEO 主域名

- Canonical 统一使用主域名。
- 兼容 WordPress core canonical、Yoast、Rank Math、AIOSEO 常见过滤器。

### 扫描与恢复

- 扫描文章/页面中的：
  - 可相对化媒体 URL
  - 可恢复的根相对媒体 URL
  - 可改写站内绝对链接
  - HTTP 混合内容
  - 源站域名/IP 暴露
- 扫描 postmeta（只读）。
- 记录最近 50 条转换日志。

### 反代与缓存检测

- 展示 HTTP_HOST、X-Forwarded-Proto、X-Real-IP、home_url、site_url、upload_baseurl 等环境。
- 检测任意 URL 响应头，重点查看 X-Cache-Status、Cache-Control、Expires。
- 生成 Nginx proxy_cache 示例配置。

## 安装

1. 上传 zip 到 WordPress 后台插件安装页面。
2. 启用插件。
3. 进入“媒体相对地址”设置页。
4. 填写允许域名、源站域名和需要的模块。
5. 先扫描预览，再执行永久修改。

## 推荐测试顺序

1. 保存设置，所有转换先保持关闭。
2. 填写允许域名和源站域名。
3. 打开“扫描 / 预览 / 修复”确认旧链接。
4. 先开启“前台临时输出”测试媒体链接。
5. 再开启“多域名访问适配”测试点击文章不跳回源站。
6. 确认稳定后，再考虑勾选文章永久转换。

## 卸载

卸载只删除插件设置和日志，不删除文章、媒体文件、数据库内容。
