# 九流媒体相对地址

WordPress 反向代理与多域名链接助手。用于处理 WordPress 在反向代理、多入口域名、源站保护、HTTPS 迁移场景下的 URL 固定问题。

## 版本

4.1.0 稳定硬化版

## 核心原则

- 默认不启用任何转换功能。
- 启用插件不会自动修改文章、媒体库、数据库或 WordPress 核心。
- 永久修改只发生在用户手动进入「扫描 / 预览 / 修复」并勾选文章执行操作时。
- 支持修订版本的文章会在批量更新前尽量创建修订。
- postmeta 只读扫描，不自动修改，避免破坏序列化数据。
- 从旧版本升级到 4.1.0 时，会重置会改变输出或写库的功能开关为关闭，但保留域名、路径、扫描范围等配置。

## 功能模块

### 媒体库地址

- 转换已经上传媒体的输出地址。
- 从开启后，新上传媒体默认相对地址输出。
- 可处理附件 URL、媒体库弹窗、特色图、srcset 等输出层。

### 文章内容地址

- 保存文章/页面时自动转换内容中的媒体地址。
- 前台显示文章时临时转换旧内容。
- 历史文章必须手动扫描、预览、勾选后执行。

### 多域名访问适配

- 站内链接跟随当前访问域名。
- 支持白名单域名模式。
- 可选任意 Host 高级模式。
- 可排除后台、登录页、REST API、Feed、Sitemap、XML 等系统端点。

### SEO Canonical

- 可将 canonical 统一到主域名。
- 支持 WordPress Core、Yoast SEO、Rank Math、AIOSEO 的常见 canonical 过滤器。

### 扫描 / 预览 / 修复

支持扫描文章正文和摘要中的：

- 媒体/静态资源绝对 URL。
- 可恢复的根相对媒体 URL。
- 可改写的站内绝对链接。
- 本站 HTTP 混合内容。
- 源站域名/IP 暴露。

支持手动执行：

- 媒体/静态资源绝对 URL → 根相对。
- 根相对媒体 URL → 完整 URL。
- 站内绝对链接 → 目标域名。
- 站内绝对链接 → 根相对。
- 本站 HTTP → HTTPS。

### 诊断工具

- 反代环境检测。
- 缓存响应头检测。
- Nginx proxy_cache 配置建议。
- 转换日志。

## 建议使用顺序

1. 备份数据库。
2. 安装并启用插件，确认所有转换开关为关闭。
3. 在设置中填写额外源站域名和多域名白名单。
4. 先使用扫描和 Dry Run 预览。
5. 先开启前台临时输出或多域名访问适配测试效果。
6. 需要永久改库时，只勾选少量文章测试。
7. 确认无误后再批量处理。

## 不做什么

- 不移动媒体文件。
- 不接管对象存储。
- 不修改 WordPress 核心。
- 不默认修改数据库。
- 不默认修改 GUID。
- 不自动修改 postmeta。
- 不自动修改 Nginx 配置。

## 安全提醒

该插件涉及文章正文、摘要、前台链接输出和多域名适配。即使插件默认安全，生产站执行永久修复前也应备份数据库并先小范围测试。
